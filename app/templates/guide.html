📖 Guide

What is MCP Gate?

MCP Gate — open-source gateway для безопасного доступа AI-агентов (LLM) к вашей инфраструктуре через SSH.

Когда вы подключаете AI-агент (Claude, ChatGPT, Cursor и др.) к вашим серверам, возникает критический вопрос безопасности: какие команды агент может выполнять, а какие — нет. MCP Gate решает эту задачу, выступая единой точкой контроля между агентами и серверами.

🛡️
Whitelist-контроль

Только явно разрешённые команды могут быть выполнены. Параметризованные команды с regex-валидацией.

🔑
Хранилище ключей

Fernet-шифрование API-ключей и токенов. Подстановка $SECRET{id} server-side. Автоматическая очистка ответов.

⏳
Workflow подтверждений

4 режима: auto, pessimistic, optimistic, strict. Уведомления в Telegram/Email. Обратный отсчёт.

🤖
Мультиагентность

Отдельный API-ключ для каждого агента. Ограничение по хостам. Rate limiting. Полный аудит по агентам.

📦
Наборы команд

Группировка команд в переиспользуемые наборы. Allow и deny типы. Привязка к хостам и агентам.

📋
Полный аудит

Каждый запрос логируется: хост, команда, агент, статус, длительность. Retention, экспорт CSV/JSON.

📦 Наборы команд: Allow и Deny

Наборы команд бывают двух типов. Они привязываются к хостам и к агентам, что позволяет гибко управлять правами доступа.

✓ Allow-набор

Список разрешённых SSH-команд. Произвольный цвет тега. Может содержать параметризованные команды с {name} плейсхолдерами.

Пример команд:
uptime
df -h
systemctl status {service}
✕ Deny-набор

Список заблокированных команд. Всегда тёмно-бордовый. Наивысший приоритет — deny всегда побеждает allow.

Пример команд:
rm -rf *
shutdown
reboot

Логика авторизации (INTERSECTION + DENY)

1. host_allow  = UNION команд всех allow-наборов хоста (enabled)
2. host_deny   = UNION команд всех deny-наборов хоста (enabled)
3. agent_allow = UNION команд всех allow-наборов агента (enabled)
4. agent_deny  = UNION команд всех deny-наборов агента (enabled)

5. effective_allow = host_allow ∩ agent_allow   ← если у агента ЕСТЬ allow-наборы
                   = host_allow                  ← если у агента НЕТ allow-наборов
6. effective_deny  = host_deny ∪ agent_deny

7. final = effective_allow − effective_deny

Команда выполняется ⟺ она входит в final

Примеры сценариев

Сценарий Результат
У агента нет наборов, у хоста allow-набор с 20 командами Агент может выполнять все 20 команд хоста
У агента allow-набор с 5 командами, у хоста — с 20 Агент может выполнять только 5 команд (пересечение)
У хоста deny-набор с reboot, у агента нет ограничений reboot заблокирована для всех агентов на этом хосте
У агента deny-набор с rm -rf * rm -rf * заблокирована для этого агента на всех хостах

🚀 Быстрый старт

  1. Bootstrap — при первом запуске генерируется SSH-ключ и MCP API ключ
  2. Добавить хост — IP/hostname, SSH user, порт
  3. Скопировать SSH-ключ на целевой сервер в ~/.ssh/authorized_keys
  4. Создать allow-набор команд — whitelist разрешённых SSH-команд
  5. Привязать набор к хосту
  6. Зарегистрировать агента — выбрать тип (Claude, Cursor и др.), сгенерировать API ключ
  7. Привязать наборы к агенту (опционально — для ограничения прав агента)
  8. Подключить агента — использовать инструкции из карточки агента
  9. Тестировать — Console в UI или прямые API-вызовы

🔗 Подключение AI-агентов

MCP-совместимые клиенты (Claude, Cursor, Windsurf, Continue, Cline)

Эти клиенты подключаются через MCP-сервер, который вызывает MCP Gate API. MCP Gate не является MCP-сервером напрямую — он выступает backend для вашего MCP-сервера.

// Пример вызова из MCP-сервера (ssh_exec tool)
POST https://your-gate-url:8090/api/exec
Headers: X-API-Key: <agent-api-key>
Body: {"host_id": "pve1", "command": "uptime", "source": "mcp"}

REST API клиенты (ChatGPT Actions, Open WebUI, custom)

// Список доступных хостов и команд
GET /api/hosts

// Выполнить команду
POST /api/exec
{"host_id": "pve1", "command": "uptime", "source": "api"}

// С параметрами
POST /api/exec
{"host_id": "rag-vm", "command": "curl -s http://grafana:3000/{path}",
 "args": {"path": "api/search"}, "source": "api"}

🔐 Модель безопасности

АутентификацияPer-agent API key (Fernet), BasicAuth для WebUI
АвторизацияINTERSECTION allow-наборов − UNION deny-наборов
СекретыFernet-шифрование, $SECRET{id} подстановка, scrub output
ПараметризацияRegex-валидация, shell-опасные символы заблокированы по умолчанию
Подтвержденияauto / pessimistic / optimistic / strict (per-host, per-command)
АудитКаждый запрос: хост, команда, агент, статус, время, длительность
Rate limitingPer-host и per-agent лимит запросов/мин
СетьLAN-only рекомендуется, Traefik + TLS для внешнего доступа
